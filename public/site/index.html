Cifrado total de datos

Todos los datos en reposo son cifrados con AES-256. Las comunicaciones utilizan TLS 1.3 exclusivamente. Las claves de cifrado son gestionadas mediante Cloud KMS de GCP, con rotación automática programada.

AES-256 TLS 1.3 Cloud KMS

Gestión de identidad y acceso

Modelo Zero Trust aplicado a cada recurso. IAM de GCP con principio de mínimo privilegio. Autenticación multifactor obligatoria. Accesos auditados y revocados automáticamente cuando ya no son necesarios.

Zero Trust IAM MFA RBAC

Aislamiento de red privada

Cada cliente opera en una VPC privada dedicada en GCP, completamente aislada de otros entornos. Firewall rules granulares, Private Google Access y Cloud NAT para tráfico saliente controlado.

VPC Privada Firewall GCP Private Access

Auditoría y trazabilidad

Cloud Audit Logs captura cada acción sobre los datos: quién accedió, desde dónde, cuándo y qué operación realizó. Los logs son inmutables y se retienen por 90 días por defecto, ampliables contractualmente.

Cloud Audit Logs SIEM Inmutable

Residencia de datos

Los datos se almacenan en la región de GCP que el cliente defina — por defecto us-central1 con opción southamerica-east1 (São Paulo) para cumplimiento de residencia en LATAM. Sin movimiento de datos no autorizado.

Data Residency LATAM Region BigQuery

Prevención de fuga de datos (DLP)

Cloud DLP de GCP inspecciona y clasifica información sensible automáticamente. Políticas de retención y eliminación de datos configurables. Acceso de producción nunca expone datos en texto claro a desarrolladores.

Cloud DLP Clasificación Data masking
